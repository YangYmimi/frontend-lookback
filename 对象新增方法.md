# Reread ES6

### Object.is()

ES6 提出 `同值相等` 算法，解决 ES5 中相等运算符号和严格相等运算符号的缺陷

```javascript
ES5
+0 === -0 //true
NaN === NaN // false

ES6
Object.is(+0, -0) // false
Object.is(NaN, NaN) // true
```

ES5 采用下面代码实现 `Object.is`

```javascript
Object.defineProperty(Object, 'is', {
  value: function(x, y) {
    if (x === y) {
      // 针对+0 不等于 -0的情况
      return x !== 0 || 1 / x === 1 / y;
    }
    // 针对NaN的情况
    return x !== x && y !== y;
  },
  configurable: true,
  enumerable: false,
  writable: true
});
```

### Object.assign()

复制所有源对象自身的`可枚举属性`, 不包括继承属性。

首个参数不能为 undefined 和 null。

其他类型的值（即数值、字符串和布尔值），会转为对象，无法转的，直接跳过，这意味着只要 undefined 和 null 不在首参数就不会报错。

但是，除了字符串会以数组形式，拷贝入目标对象，其他值都不会产生效果。

(1). 浅拷贝: *浅拷贝(如果源对象某个属性的值是对象，那么目标对象拷贝得到的是这个对象的引用)*

```javascript
const obj1 = {a: {b: 1}};
const obj2 = Object.assign({}, obj1);

obj1.a.b = 2;
obj2.a.b // 2
```

(2). 同名属性直接替换，而非合并

```javascript
const target = { a: { b: 'c', d: 'e' } }
const source = { a: { b: 'hello' } }
Object.assign(target, source)
// { a: { b: 'hello' } }
```

(3). 数组当作对象处理

```javascript
Object.assign([1, 2, 3], [4, 5])
=>
[4, 5, 3]
```

(4). 复制函数的值

若复制的内容是一个函数，则是复制的函数调用的结果，不会拷贝赋值方法或者取值方法

### Object.getOwnPropertyDescriptors()

返回指定对象所有自身属性（非继承属性）的描述对象。

```javascript
function getOwnPropertyDescriptors(obj) {
  const result = {};
  for (let key of Reflect.ownKeys(obj)) {
    result[key] = Object.getOwnPropertyDescriptor(obj, key);
  }
  return result;
}
```

`Reflect.ownKeys` 用于返回对象的所有属性。基本等同于 `Object.getOwnPropertyNames` 与 `Object.getOwnPropertySymbols` 之和

该方法的引入目的，主要是为了解决 `Object.assign()` 无法正确拷贝 `get属性` 和 `set属性` 的问题。
